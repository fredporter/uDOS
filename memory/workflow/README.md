# Memory Workflow Structure

This folder contains your mission-based workflow system.

## Structure

### active/
Missions currently in progress. Each mission can have milestones and action items.

### completed/
Recently finished missions. Review outcomes before archiving to legacy.

### archived/
Historical missions no longer actively referenced but preserved for context.

---

## Workflow

1. **Start a mission** → Create folder in `active/`
2. **Complete milestones** → Document in mission folder
3. **Finish mission** → Move to `completed/`
4. **Archive mission** → Extract outcomes to `/memory/legacy/`, move to `archived/`

---

See `/memory/README.md` for complete workflow documentation.
