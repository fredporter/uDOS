## Technical Details

### Architecture
```
User → SVGHandler → SVGGenerator → [Gemini API OR Template] → Validate → Save
```

### Key Classes
- **SVGGenerator** - Main service class
  - `generate()` - Public API
  - `_build_prompt()` - AI prompt construction
  - `_generate_with_ai()` - Gemini integration with fallback
  - `_generate_template()` - Template fallback
  - `_validate_svg()` - XML validation
  - `_post_process()` - xmlns/viewBox injection
  - `_save_svg()` - File output

- **SVGHandler** - Command handler
  - `handle_command()` - Parse args, call generator
  - `display_help()` - Usage information

### Dependencies
- **Required**: None (templates work without API)
- **Optional**: Gemini API (via ok_assistant extension)
- **Python**: 3.10+ (type hints)

## Lessons Learned

### What Worked Well
1. **Incremental development** - 3 moves, clear steps
2. **Template fallback** - Offline capability from start
3. **Defensive validation** - Validate at multiple points
4. **String-based processing** - Simpler than XML parsing
5. **Comprehensive testing** - Caught issues early
6. **Mocking in tests** - Fast execution without API

### What Could Improve
1. **AI integration** - Initially called wrong method
2. **Type checking** - Needed isinstance() for colors
3. **Validation timing** - Moved to after post-processing
4. **Test speed** - Initially tried AI, now use templates

### Best Practices Confirmed
1. **Always validate AI output** - Don't trust raw responses
2. **Provide fallbacks** - Templates ensure offline works
3. **Test without external dependencies** - Mock or disable
4. **Document as you build** - Easier than after
5. **Break into moves** - Manageable chunks
