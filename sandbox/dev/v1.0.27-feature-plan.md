## Philosophy: Core vs Extensions

### Core Features (v1.0.27)
**Criteria**:
- ✅ Fully implemented and tested
- ✅ Essential for text-first computing workflow
- ✅ CLI-based, no complex dependencies
- ✅ Stable, predictable behavior
- ✅ Part of daily uDOS usage

### Extension Features (v1.1.0+)
**Criteria**:
- ⚠️ Incomplete implementation or stubs
- ⚠️ Experimental or game-like mechanics
- ⚠️ Complex systems requiring more dev time
- ⚠️ Optional enhancements, not core workflow
- ⚠️ Web-based or server-dependent features

## Notes

**Philosophy Reminder**:
uDOS v1.0.27 should be a **rock-solid text-first computing CLI** with:
- Predictable file operations
- Reliable AI assistance
- Stable system management
- Optional knowledge tools
- Zero game mechanics in core

Game/exploration features are awesome but belong in extensions where users can opt-in.

**Backward Compatibility**:
- All v1.0.26 features remain
- No breaking changes to existing scripts
- Deferred commands still in commands.json (just marked)
- Can still access via ucode if needed

**Extension Strategy**:
- `/extensions/core/` - Production extensions (teletext, dashboard)
- `/extensions/game-mode/` - Game features (MAP, scenarios, survival)
- `/extensions/experimental/` - WIP features
- Extensions disabled by default, enabled via `POKE` or config
