## Command Reference

### Mission Commands
```bash
MISSION CREATE <name> --steps <steps> --priority <level>
MISSION START <mission-id>
MISSION PAUSE <mission-id>
MISSION RESUME <mission-id>
MISSION COMPLETE <mission-id>
MISSION STATUS [mission-id]
MISSION REMOVE <mission-id>
MISSION LIST [--status <filter>]
MISSION CHECKPOINT <mission-id> [--restore]
```

### Schedule Commands
```bash
SCHEDULE CREATE <name> <pattern> <command>
SCHEDULE LIST [--enabled | --disabled]
SCHEDULE STATUS <task-id>
SCHEDULE ENABLE <task-id>
SCHEDULE DISABLE <task-id>
SCHEDULE REMOVE <task-id>
SCHEDULE RUN <task-id>
```

### Workflow Commands
```bash
WORKFLOW CREATE <name> --template <template-name>
WORKFLOW VALIDATE <workflow-file>
WORKFLOW RUN <workflow-file>
WORKFLOW STATUS
WORKFLOW TEMPLATES
```

### Resource Commands
```bash
RESOURCE STATUS
RESOURCE QUOTA [provider]
RESOURCE ALLOCATE <mission-id> --api <calls> --disk <mb> --priority <level>
RESOURCE RELEASE <mission-id>
RESOURCE THROTTLE [mission-id]
RESOURCE SUMMARY
```

## Architecture Highlights

### Service Layer
```
core/services/
├── mission_manager.py      # Mission lifecycle & execution
├── scheduler.py            # Time-based task scheduling
├── resource_manager.py     # Resource tracking & allocation
└── output_pacer.py         # Organic output rendering
```

### Command Layer
```
core/commands/
├── workflow_handler.py     # Workflow & mission commands
└── resource_handler.py     # Resource monitoring commands
```

### Extension Layer
```
extensions/core/mission-control/
├── extension.json          # Extension manifest
├── dashboard.html          # Web UI
├── dashboard.css           # Styling
├── dashboard.js            # Frontend logic
└── dashboard_handler.py    # Flask API + WebSocket
```

### Data Flow
```
User Command
    ↓
Command Handler (workflow_handler.py)
    ↓
Service Layer (mission_manager.py, resource_manager.py)
    ↓
State Persistence (sandbox/user/*.json)
    ↓
WebSocket Broadcast (dashboard_handler.py)
    ↓
Dashboard Update (dashboard.js)
```

## Performance Metrics

### Resource Management
- API Quotas: Gemini (1500/day), GitHub (5000/hour)
- Disk Monitoring: Real-time via psutil
- CPU/Memory: Cached for 5 seconds
- Allocation: Priority-based with preemption

### Output Pacing
- Typing Speed: 40 chars/sec (configurable)
- Speed Ramping: 50% → 150% → 50% (quadratic easing)
- Viewport Threshold: 80% full → pause
- Pause Durations: 200ms (short), 350ms (medium), 500ms (long)

### Dashboard
- Auto-Refresh: 5 seconds (when WebSocket disconnected)
- WebSocket Reconnect: 5 seconds delay
- Timeline Limit: 50 events (configurable)
- Celebration Duration: 3 seconds
- Asset Size: ~37KB total (~10KB gzipped)

- Progress bars for long-running steps

### All Systems → Dashboard

- REST API for data fetching
- Live progress monitoring
- Celebration on mission completion
