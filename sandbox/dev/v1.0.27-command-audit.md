## Critical Routing Issues Found

### 1. **PANEL Command - Module Not Routed** ❌
- **Symptom**: `💀 UNKNOWN CHAMBER: UI`
- **Cause**: PANEL command uses `[UI|PANEL*...]` in commands.json
- **Problem**: `uDOS_commands.py` has NO route for `module == "UI"`
- **Handler Exists**: `self.panel_handler = PanelCommandHandler()` is created
- **Fix Required**: Add UI module route in `handle_command()` method

### 2. **SPLASH Command - Handler Missing** ❌
- **Symptom**: `<ERROR_UNKNOWN_SYSTEM_COMMAND>`
- **Cause**: SPLASH routes to `[SYSTEM|SPLASH*...]`
- **Problem**: `SystemCommandHandler.handle()` has NO 'SPLASH' mapping
- **Commands.json**: Fully defined with LOGO, TEXT, FILE modes
- **Fix Required**: Implement `handle_splash()` or remove from commands.json

### 3. **FILE Command - Unknown** ❌
- **Symptom**: `<ERROR_UNKNOWN_SYSTEM_COMMAND>` when typed alone
- **Cause**: User typing "FILE" without subcommand (FILE PICK, FILE RECENT, etc.)
- **Issue**: No fallback handler for bare "FILE" command
- **Note**: May be user error, not a bug - needs documentation
