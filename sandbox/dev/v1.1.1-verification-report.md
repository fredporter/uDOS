## Issues Found

### Issue 1: SET Bracket Notation Not Implemented

**Severity**: Medium
**Impact**: Users cannot use modern `SET[var = value]` syntax

**Example**:
```uscript
SET[count = 42]         # ❌ Fails with syntax error
SET count = 42          # ✅ Works (old syntax)
```

**Error Message**:
```
❌ SET syntax: SET varname = value [PERSISTENT]
```

**Root Cause**: The `_normalize_modern_syntax()` method in `ucode.py` does not handle SET bracket notation.

**Fix Needed**: Add bracket normalization for SET command similar to PRINT.

---

### Issue 2: Escape Sequence Processing

**Severity**: Low
**Impact**: Escaped quotes appear literally in output

**Example**:
```uscript
PRINT "He said \"Hello\""   # Expected: He said "Hello"
                             # Actual:   He said \"Hello\"
```

**Root Cause**: The `_handle_print()` method removes outer quotes but does not unescape inner content.

**Fix Needed**: Add escape sequence processing after quote removal:
```python
text = content[1:-1]  # Remove quotes
text = text.replace(r'\"', '"')  # Unescape quotes
text = text.replace(r'\n', '\n')  # Unescape newlines
text = text.replace(r'\t', '\t')  # Unescape tabs
```

---

### Issue 3: Script Execution via uDOS.py

**Severity**: High
**Impact**: Cannot run .uscript files directly

**Example**:
```bash
python3 uDOS.py script.uscript    # ❌ Parsed as old commands, not uCODE
./start_udos.sh script.uscript    # ❌ Same issue
```

**Current Workaround**: Use uCODE interpreter directly in Python:
```python
from core.interpreters.ucode import UCodeInterpreter
interp = UCodeInterpreter()
result = interp.execute_file('script.uscript')
```

**Root Cause**: When uDOS.py receives a `.uscript` file, it uses the old command parser instead of the uCODE interpreter.

**Fix Needed**: Update `uDOS_main.py` to detect `.uscript` extension and route to uCODE interpreter:
```python
if script_file.endswith('.uscript'):
    from core.interpreters.ucode import UCodeInterpreter
    interp = UCodeInterpreter()
    result = interp.execute_file(script_file)
else:
    # Old command parsing
    ...
```

## Recommendations

### Priority 1: Fix Script Execution (Critical)

**Why**: Users cannot run .uscript files as documented
**Effort**: 30 minutes
**Files**: `core/uDOS_main.py`

Add uCODE interpreter routing for `.uscript` files:
1. Check file extension
2. If `.uscript`, use `UCodeInterpreter.execute_file()`
3. If other, use old command parser

### Priority 2: Implement SET Bracket Notation (Medium)

**Why**: Modern syntax incomplete without it
**Effort**: 1-2 hours
**Files**: `core/interpreters/ucode.py`

Add bracket normalization in `_normalize_modern_syntax()`:
1. Detect `SET[...] ` or `SET [...]` or `[SET|...]`
2. Extract content between brackets/pipes
3. Convert to `SET ...` format
4. Pass to existing `_handle_set()` method

### Priority 3: Fix Escape Sequences (Low)

**Why**: Improves string handling
**Effort**: 15 minutes
**Files**: `core/interpreters/ucode.py`

Add escape processing in `_handle_print()`:
1. After removing outer quotes
2. Replace `\"` with `"`
3. Replace `\n` with newline
4. Replace `\t` with tab
5. Replace `\\` with `\`

### Optional: Update Documentation

Update wiki with workarounds:
- Document SET bracket limitation
- Show Python API usage for script execution
- Add escape sequence examples
