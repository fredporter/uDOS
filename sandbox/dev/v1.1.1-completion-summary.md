## 📚 Modern Syntax Reference

### Commands

```ucode
# PRINT - Multiple formats (all equivalent)
PRINT "Hello World"
PRINT[Hello World]
PRINT [Hello World]
[PRINT|Hello World]

# Template strings
PRINT "Hello ${name}, you have ${count} items"
PRINT[System: ${status}]

# SET - Assignment with brackets
SET username = "Alice"
SET[age = 25]
SET [count = 100]
[SET|status = active]

# GET - Retrieval with brackets
GET username
GET[age]
GET [count]
[GET|status]

# One-line IF with THEN
IF x > 5 THEN PRINT "x is large"
IF{x > 5} THEN PRINT[x is large]
IF{status == "ready"} THEN SET[active = true]
```

### Syntax Features

**Bracket Notation**:
- Square brackets `[]` for commands (PRINT, SET, GET)
- Curly braces `{}` for IF conditions
- Three equivalent formats all normalize to same internal representation

**Template Strings**:
- Use `${var}` for variable substitution
- Works in PRINT commands and quoted strings
- Replaces old concatenation with `+`

**Context-Sensitive Normalization**:
- Reserved chars: `~^-+|<>*`
- SET allows `=` (assignment operator)
- IF doesn't normalize (preserves comparison operators)

### Design Decisions

1. **Curly Braces for IF**: User requested `IF{condition}` instead of `IF[condition]`
   - Separates condition syntax from command brackets
   - More intuitive for users familiar with other languages

2. **Three Bracket Formats**: All equivalent, user choice
   - `PRINT[text]` - Compact, modern
   - `PRINT [text]` - Spaced, readable
   - `[PRINT|text]` - Traditional uCODE pipe syntax
   - All normalize to `PRINT "text"` internally

3. **Template Strings**: `${var}` instead of concatenation
   - Cleaner syntax: `PRINT "Hello ${name}"` vs `PRINT "Hello " + name`
   - Matches modern language conventions (JavaScript, Shell, etc.)

4. **Context-Sensitive Reserved Chars**:
   - SET allows `=` for assignment
   - IF doesn't normalize (needs operators)
   - Prevents breaking valid syntax

5. **No Backward Compatibility**: Per user directive
   - Skip migration tools
   - Update system templates directly
   - Focus on modern syntax adoption
