## Bug Fix: SystemCommandHandler Theme Attribute Error

**Date**: November 22, 2025  
**Issue**: `'SystemCommandHandler' object has no attribute 'theme'`

### Problem
Commands STATUS, SETTINGS, THEME, VIEWPORT, PALETTE, CONFIG, DASHBOARD, and REPAIR were failing with AttributeError when trying to delegate to specialized handlers.

### Root Cause
`BaseCommandHandler.__init__()` was loading theme data and storing `self.lexicon` and `self.messages`, but was NOT storing the `theme` parameter itself as `self.theme`. 

`SystemCommandHandler` methods were trying to pass `theme=self.theme` when creating specialized handlers like:
- DashboardHandler
- ConfigurationHandler  
- RepairHandler

### Solution
Added `self.theme = theme` to `BaseCommandHandler.__init__()` before loading theme data.

**File Modified**: `core/commands/base_handler.py`

```python
def __init__(self, theme='dungeon', **kwargs):
    # Store theme name
    self.theme = theme  # ← Added this line
    
    # Load merged theme...
    theme_data = load_theme(theme, root_path=Path(__file__).parent.parent)
    ...
```

```bash
# After fix:

```
