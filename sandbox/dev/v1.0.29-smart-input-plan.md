## 🎯 Core Philosophy

**All commands should work with just ONE WORD and prompt for options**

Current problems:
- Commands like `CONFIG GET API_KEY` require too many words
- `SETUP` wizard exists but commands don't leverage it
- `SYSTEM|PROMPT` and `SYSTEM|GET` exist in ucode but aren't exposed
- Story.json configuration is incomplete (user setup flow broken)
- No unified input field system

**v1.0.29 Goal**: Every command becomes conversational
- Type `CONFIG` → Smart prompt asks "What would you like to configure?"
- Type `SETUP` → Interactive wizard with proper story setup
- Type `FILE` → Smart prompt asks "Which file operation?"
- Type `BANK` → Smart prompt asks "Search, list, or browse?"

```python
class InputManager:
    """Unified input system for all uDOS commands"""

    def prompt_user(self, message: str, default: str = None, required: bool = False):

        """GET field from nested dict (story.json, user.json, etc.)"""

        """SET field in nested dict with auto-save"""
```

### Phase 3: GET System Implementation

**3.1 Expose GET/SET to ucode and commands**

Currently `SYSTEM|GET_FIELD` exists in setup.uscript but isn't accessible

**Solution**: Make GET a first-class command

```python
# In core/commands/system_handler.py

def handle_get(self, params, grid, parser):
    """
    GET field value from story/config/system

    Usage:
        GET                          → Interactive field picker
        GET STORY.USER_NAME          → Get story field
        GET CONFIG.GEMINI_API_KEY    → Get config field
        GET SYSTEM.THEME             → Get system setting
        GET WORLDMAP.CITIES.Sydney   → Get nested data
    """

def handle_set(self, params, grid, parser):
    """
    SET field value in story/config/system

    Usage:
        SET                                    → Interactive setter
        SET STORY.USER_NAME "Fred"             → Set story field
        SET CONFIG.OFFLINE_MODE true           → Set config
        SET SYSTEM.THEME dungeon               → Set system setting
    """
```

**Add to commands.json**:
```json
{
  "NAME": "GET",
  "SYNTAX": "GET [field.path]",
  "DESCRIPTION": "Get configuration field value interactively",
  "UCODE_TEMPLATE": "[SYSTEM|GET*$1]",
  "DEFAULT_PARAMS": {"$1": ""}
},
{
  "NAME": "SET",
  "SYNTAX": "SET [field.path] [value]",
  "DESCRIPTION": "Set configuration field value interactively",
  "UCODE_TEMPLATE": "[SYSTEM|SET*$1*$2]",
  "DEFAULT_PARAMS": {"$1": "", "$2": ""}
}
```

## 💡 Key Design Decisions

### 1. Backward Compatibility
**ALL existing command syntax must still work**
- `CONFIG GET API_KEY` still works (explicit mode)
- `CONFIG` alone triggers smart mode (interactive)
- This is non-breaking for existing .uscript files

### 2. Smart vs Explicit
**Users can choose their preferred style**
- Interactive users: Single word commands
- Script writers: Full explicit syntax
- Both are first-class citizens

### 3. Progressive Disclosure
**Show options when needed, not all at once**
- Initial prompt shows 3-5 top options
- "More options" choice reveals advanced features
- Common tasks are one or two prompts away

### 4. Sensible Defaults
**Every prompt should have a good default**
- Timezone: Auto-detect
- Theme: dungeon (most tested)
- Location: Skip (optional)
- API keys: Skip (optional)
