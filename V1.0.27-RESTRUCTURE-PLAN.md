# v1.0.27 Stable Release - Restructuring Plan

**Date**: November 19, 2025
**Goal**: Finalize v1.0.27 as stable release, prepare for v1.1.0+ development

## Overview

Transform uDOS from development state to production-ready distribution with clean separation between:
- **Distribution content** (git tracked, read-only for users)
- **Development workspace** (/dev folder, local-only for contributors)
- **User workspace** (/memory folder, user-generated content)

---

## Phase 1: Documentation Audit & Migration

### /docs → Wiki Migration
**Goal**: Extract useful content, archive the rest

#### Priority Content for Wiki:
1. **Command references** → Already in wiki (Command-Reference.md)
2. **Quick references** → Consolidate into wiki pages
3. **Architecture docs** → Update wiki/Architecture.md
4. **Release notes** → Keep for historical reference

#### Archive Locally (keep in /dev):
- Planning documents (v1.0.26-PLAN.md, etc.)
- Development summaries (SESSION-SUMMARY-*.md)
- Phase completion reports
- Validation reports

#### Delete/Consolidate:
- Duplicate quick references
- Outdated roadmap fragments
- Development scratch files

## Phase 2: Memory Folder Structure

### Keep in Git (Distribution Template):
```
memory/

│   └── .gitkeep
```

### Exclude from Git (User-Generated):
- knowledge.db
- xp.db
- logs/
- private/
- personal/
- groups/
- missions/
- scenarios/
- workspace/ (user files)
- sandbox/ (user files)

## Phase 3: /docs → /dev Restructuring

### New /dev Folder Structure:
```
dev/
├── README.md             # Development guide
├── planning/             # GIT TRACKED
│   ├── roadmap/         # v1.1.0+ roadmaps
│   ├── features/        # Feature specifications
│   └── releases/        # Release planning
├── archive/              # LOCAL ONLY
│   ├── v1.0.x/          # All v1.0 dev notes
│   │   ├── sessions/
│   │   ├── phases/
│   │   └── summaries/
│   └── historical/      # Old docs
├── notes/                # LOCAL ONLY
│   ├── scratch/         # Developer scratch work
│   └── ideas/           # Unformed concepts
└── tools/                # GIT TRACKED (optional)
    └── scripts/         # Development utilities
```

### Git Tracking Strategy:
- `/dev/planning/` → Tracked (roadmaps, specs)
- `/dev/archive/` → NOT tracked (local dev history)
- `/dev/notes/` → NOT tracked (personal notes)
- `/dev/tools/` → Tracked if useful for contributors
