# 🗄️ uDOS Smart Backup System v1.3

The Smart Backup System provides intelligent, role-based backup management with encryption and automatic retention.

## 🎯 Key Features

### 🔐 **Role-Based Storage**
- Backups are stored in the user's role-specific directory
- User role is read from `sandbox/user.md`
- Default mapping: unknown roles → `ghost/backup/`

**Role Mapping:**
- `tomb` → `tomb/backup/`
- `sorcerer` → `sorcerer/backup/`
- `wizard` → `wizard/backup/`
- `imp` → `imp/backup/`
- `drone` → `drone/backup/`
- `ghost` → `ghost/backup/`
- `user` → `ghost/backup/` (default)

### 🔒 **Automatic Encryption**
- Encrypts backups if user has password set in `user.md`
- Uses first 32 characters of password hash as encryption key
- AES-256-CBC encryption with PBKDF2 (10,000 iterations)
- Encrypted files have `.enc` extension

### 📦 **Intelligent Retention**
- Maintains exactly **5 backups maximum**
- Keeps **4 latest backups** by modification time
- Keeps **1 largest backup** for space efficiency
- Automatic cleanup on each backup run

### 🏷️ **Hex Timestamp Naming**
- Format: `YYYYMMDD-HHMMSS-backup.tar.gz[.enc]`
- Example: `013500C4-03448F-backup.tar.gz.enc`
- Ensures chronological sorting and uniqueness

## 📁 Backup Content

The system backs up:
- **uMEMORY/** - Complete user memory system
- **sandbox/** - User workspace and files

Excludes:
- `.DS_Store` files
- `*.log` files  
- `cache/*` directories

## 🔍 Troubleshooting

### Common Issues

**Backup not encrypted:**
- Check if user has password set: `**Password**: SET`
- Verify password hash exists in `user.md`

**Wrong backup location:**
- Check user role in `sandbox/user.md`
- Verify role directory exists

**Too many backups:**
- Cleanup runs automatically
- Manual cleanup: Delete oldest files keeping 4 newest + 1 largest

### File Structure Check
```bash
# Check backup directory
ls -la /path/to/role/backup/

# Check user configuration
cat sandbox/user.md

# Test encryption key extraction
grep "**Password Hash**:" sandbox/user.md | sed 's/^\*\*Password Hash\*\*: *//' | cut -c1-32
```
