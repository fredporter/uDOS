### 5. DIAGRAM GENERATE Command (300+ lines)
**File:** `core/commands/diagram_handler.py` (extended)

**New Methods:**
- `_generate(args)` - Main entry point, argument parsing
- `_generate_single(source_path, options)` - Process one file (5 stages)
- `_generate_batch(directory, options)` - Process entire directories
- `_generate_help()` - Comprehensive usage documentation

**Multi-Stage Pipeline:**
```
Stage A: Source Analysis
  └─> Load source document, extract metadata

Stage B: Web Crawl (optional)
  └─> PEEK supplementary content via URLs

Stage C: Text Generation
  └─> Gemini API: source + crawled → Markdown + citations

Stage D: SVG Generation
  └─> Gemini API: topic → Technical-Kinetic diagram

Stage E: Final Assembly
  └─> Save .md + .svg + .meta.json, validate quality
```

**Command Syntax:**
```bash
DIAGRAM GENERATE <source_file> [options]

Options:
  --crawl              Enable PEEK web crawler
  --style <type>       SVG style: technical|kinetic
  --output <dir>       Output directory
  --citations-strict   Require 100% coverage
  --validate          Run full quality checks
  --format <types>     Output: md,svg,ascii,teletext
  --batch <dir>        Process directory
```
