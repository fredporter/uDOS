## Technical Specifications

### ASCII Art Generator (C64 PETSCII)
**Method:** `gemini_generator.generate_ascii()`  
**Configuration:**
- Character set: C64 PETSCII (box drawing, line art)
- Dimensions: 80 columns × 24 lines (terminal standard)
- Output format: Plain text UTF-8 (.ascii extension)
- Validation: Width/height limits, within_limits flag

**Prompt Template:**
```
Use C64 PETSCII character set
Maximum width: 80 characters
Maximum height: 24 lines
Use box drawing characters: ┌─┐│└┘├┤┬┴┼
Include labels and annotations
```

### Teletext Generator (WST Mosaic)
**Method:** `gemini_generator.generate_teletext()`  
**Configuration:**
- Format: WST Level 1 (World System Teletext)
- Grid: 40 columns × 25 rows
- Palette: 8 colors (black, red, green, yellow, blue, magenta, cyan, white)
- Mosaic cells: 2×3 pixel blocks
- Output: HTML with inline styles (.html extension)

**Prompt Template:**
```
WST 8-color palette: black, red, green, yellow, blue, magenta, cyan, white
40×25 character grid
Mosaic blocks (2×3 pixel cells)
Use Level 1 compatibility
Output as HTML with inline styles
```

### HTML Output Structure
**Teletext HTML Features:**
- Custom `@font-face` with Teletext font
- 40-character monospace grid
- CRT screen effects (glow, border)
- WST color classes (.c_red, .c_green, etc.)
- Responsive layout with centered display

## Integration with Existing Pipeline

### Multi-Stage Processing
The `DIAGRAM GENERATE` command now supports all formats in a single execution:

**Stage A:** Source Analysis (Markdown input)  
**Stage B:** Web Crawl (optional PEEK enrichment)  
**Stage C:** Text Generation (Gemini → Markdown with citations)  
**Stage D:** Asset Generation (now 3 sub-stages):
- D1: SVG (Technical-Kinetic diagrams)
- D2: ASCII (Terminal art)
- D3: Teletext (Retro web graphics)

**Stage E:** Final Assembly (save all requested formats)

### Output Files
For source `water_purification.md`:
```
knowledge/generated/
├── water_purification.md          (Markdown with citations)
├── water_purification.svg         (Technical-Kinetic diagram)
├── water_purification.ascii       (C64 PETSCII flowchart)
├── water_purification.html        (WST Teletext graphics)
└── water_purification.meta.json   (Metadata for all formats)
```

## Known Issues & Improvements

### ASCII Width Overflow
**Issue:** Some generated ASCII art exceeds 80-column limit (observed 57-64 cols but content wraps)  
**Impact:** Terminal display may wrap lines  
**Fix:** Enhance prompt to enforce strict 80-column limit with padding

**Resolution Plan:**
```python
# Add to ascii_generation template:
"CRITICAL: EVERY LINE must be ≤80 characters
Pad short lines with spaces to align boxes
Test each line length before output"
```

### Teletext Font Loading
**Issue:** Custom Teletext font loaded from external URL (GitHub)  
**Impact:** Requires internet connection, may fail offline  
**Fix:** Include fallback monospace fonts (implemented)

**Future Enhancement:** Bundle Teletext font locally in `extensions/assets/fonts/`

### SVG Token Limits
**Issue:** Complex diagrams can exceed Gemini token limits, causing truncation  
**Impact:** Occasional SVG validation errors  
**Fix:** Implement diagram complexity scoring and split large diagrams

**Mitigation:** Current prompts emphasize simplicity ("clean geometry", "minimal paths")
