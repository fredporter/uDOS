## 🎯 Core Problem Analysis

### Current State Issues

1. **Inconsistent Input Methods**
   - Mix of `input()`, `prompt_toolkit.prompt()`, and custom implementations
   - 35+ different input patterns across codebase
   - No standardization for option selection
   - Micro editor uses basic `input()` (15 instances)
   - Knowledge picker uses basic `input()` (5 instances)
   - Theme builder uses basic `input()` (15+ instances)

2. **Limited Visual Feedback**
   - Teletext UI exists but not integrated everywhere
   - Block selectors not used consistently
   - Arrow key navigation only in smart_prompt
   - No visual indication of selected options

3. **prompt_toolkit Limitations**
   - **Terminal Dependency**: Works differently on macOS/Linux/Windows
   - **ANSI Issues**: Some terminals don't support full feature set
   - **Mouse Support**: Unreliable, disabled in fallback mode
   - **Copy-Paste**: Interferes with terminal scrollback
   - **Performance**: Can be slow on older terminals
   - **Complexity**: Requires extensive error handling

4. **User Experience Gaps**
   - No predictive typing visible while user types
   - Commands autocomplete but options don't
   - File paths not autocompleted
   - No visual preview of selections
   - 1-9 keyboard shortcuts underutilized

## 🔍 CLI Technical Constraints

### What Works Reliably

✅ **Basic Text Input/Output**
- `print()` - 100% reliable
- `input()` - 100% reliable
- ANSI colors (if terminal supports)
- Unicode box-drawing characters

✅ **prompt_toolkit Features (when working)**
- History navigation (↑↓)
- Basic autocomplete
- Tab completion
- Ctrl+C/Ctrl+D handling

### What's Unreliable

❌ **Advanced Features**
- Mouse events (platform-dependent)
- Complex key bindings (terminal-specific)
- Simultaneous multi-column displays
- Real-time updates without flicker
- Inline editing of previous lines

### Best Practice Approach

**Principle**: Use the simplest method that works reliably

1. **For simple choices**: Numbered menu + basic `input()`
2. **For text input**: `prompt_toolkit` with fallback to `input()`
3. **For autocomplete**: Show suggestions, don't force them
4. **For navigation**: ↑↓←→ keys via prompt_toolkit, numbers (1-9) as fallback

## 🎯 Key Design Decisions

### 1. Always Have a Fallback

**Principle**: Every advanced feature must work without it

```python
def select_option(self, title, options):
    if self.advanced_mode:
        try:
            return self._select_with_arrows(title, options)
        except Exception:
            self.advanced_mode = False  # Permanent fallback

    # Fallback: numbered menu
    return self._select_with_numbers(title, options)
```

### 2. Numbers Over Arrows

**Why**: Numbers (1-9) work in ALL terminals, arrow keys don't

```
┌────────────────────┐
│ Select Theme       │
├────────────────────┤
│ 1. ► Dungeon       │  ← Use ► to show selection
│ 2.   Science       │  ← Type '2' to select
│ 3.   Cyberpunk     │  ← Press ↓ if supported
└────────────────────┘

Type number or use arrows: █
```

### 3. Show, Don't Force

**Principle**: Show suggestions, but allow free typing

```
> FILE █

Suggestions:
  1. FILE EDIT readme.md
  2. FILE SHOW config.json
  3. FILE PICK

Press 1-3 for suggestion, or continue typing...
```

### 4. Visual Feedback Without Flicker

**Approach**: Render complete menu once, don't update in real-time

```python
# GOOD: Render once, wait for input
print(render_menu(items, selected=0))
choice = input("> ")

# BAD: Constant re-rendering (flickers)
while True:
    clear_screen()  # ❌ Causes flicker
    print(render_menu(items, selected))
    key = get_key()
```

### Numbered Menu Renderer

```python
def render_numbered_menu(
    self,
    title: str,
    items: List[str],
    selected_index: int = 0
) -> str:
    """Render menu with numbers and selection indicator"""
    output = []

    # Title
    output.append(f"\n┌{'─' * (len(title) + 2)}┐")
    output.append(f"│ {title} │")
    output.append(f"├{'─' * (len(title) + 2)}┤")

    # Items (limit to 9)
    for i, item in enumerate(items[:9], 1):
        indicator = '►' if i-1 == selected_index else ' '
        output.append(f"│ {i}. {indicator} {item}")

    output.append(f"└{'─' * (len(title) + 2)}┘")

    return '\n'.join(output)
```

### Smart Option Parser

```python
def parse_selection(
    self,
    user_input: str,
    options: List[str]
) -> Tuple[int, str]:
    """Parse user selection (number, text, or partial match)"""

    # Try numeric
    if user_input.isdigit():
        index = int(user_input) - 1
        if 0 <= index < len(options):
            return index, options[index]

    # Try exact match
    for i, option in enumerate(options):
        if option.lower() == user_input.lower():
            return i, option

    # Try partial match
    matches = [
        (i, opt) for i, opt in enumerate(options)
        if user_input.lower() in opt.lower()
    ]

    if len(matches) == 1:
        return matches[0]

    raise ValueError("Ambiguous or invalid selection")
```
