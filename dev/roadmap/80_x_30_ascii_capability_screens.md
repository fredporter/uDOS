# 80×30 ASCII Capability — Pattern & Animation Index

This document intentionally **does NOT repeat full 80×30 frames**.
Instead, it records the **canonical character combinations**, motifs, and animation deltas used to construct frames procedurally.

All examples below are **representative rows or micro-blocks**, not full screens.

---

## GLOBAL ASSUMPTIONS
- Framebuffer target: 80 columns × 30 rows
- Monospaced terminal
- Full frames are generated by repetition, tiling, or offsetting these primitives
- Animations are defined by **delta rules**, not stored frames

## TELETEXT BLOCK GRAPHIC PATTERNS

This section provides **Teletext-style mosaic primitives**, a **canonical sextant table**, and a **pattern cookbook**.
We store **rules + masks**, not full 80×30 frames.

---

### A) Fallback block alphabet (widely supported)
**Full/half blocks**
```
█  ▓  ▒  ░
▀  ▄
▌  ▐
```

**Quadrants (2×2 micro-tiles)**
```
▘ ▝ ▖ ▗
▙ ▟ ▛ ▜
```

**UI lines (box drawing)**
```
─ │ ┌ ┐ └ ┘ ├ ┤ ┬ ┴ ┼
```

---

### B) Sextant model (2×3 micro-cells)
Teletext mosaic graphics behave like **2 columns × 3 rows** per character cell.
Represent as a 6‑bit mask:

```
[ b0 b1 ]   top row
[ b2 b3 ]   mid row
[ b4 b5 ]   bot row
```

Legend used in diagrams:
- `#` = ON (filled)
- `.` = OFF (empty)

### D) Pattern cookbook (procedural, Teletext-first)
Each pattern is specified as:
- `PAT-ID`
- `ALPHABET` (characters used)
- `TILE` (repeat unit)
- `FILL` (how to tile into 80×30)
- `ANIM` (Δ/PHASE rules)
- `PALETTE` (uDOS indices)

#### PAT-001 — Checker Mosaic
ALPHABET: `█ ░`
TILE:
```
█░
░█
```
FILL: repeat tile across rows/cols
ANIM: PHASE flip (swap `█` and `░`) every 1–2 frames
PALETTE: `█`→ idx 3 (White) or idx 2 (Light Grey); `░`→ idx 1 (Dark Grey)

#### PAT-002 — Brickwork
ALPHABET: `█ ▓`
TILE:
```
██▓▓
▓▓██
```
ANIM: Δx +1 every frame (wrap)
PALETTE: `█`→ idx 2 (Light Grey); `▓`→ idx 1 (Dark Grey)

#### PAT-003 — Scanline Shimmer
ALPHABET: `▓ ░`
TILE:
```
▓▓▓▓▓▓▓▓
░░░░░░░░
```
ANIM: Δy +1 (vertical roll)
PALETTE: `▓`→ idx 2 (Light Grey); `░`→ idx 1 (Dark Grey)

#### PAT-004 — Interference Bands
ALPHABET: `- = # @ % X W M B`
TILE (rows):
```
..--..--..--
==..==..==..
##..##..##..
@@..@@..@@..
```
ANIM: rotate band list (PHASE)
PALETTE: map density to greyscale ramp (idx 16–23)

#### PAT-005 — Sextant Gradient (true Teletext feel)
ALPHABET: sextant masks `SXT-00 … SXT-3F`
FILL: choose masks by density threshold (e.g. 0%, 16%, 33%, 50%, 66%, 83%, 100%)
ANIM: PHASE shift thresholds for a “sparkle”
PALETTE: tie density to greyscale (idx 16–23) or terrain (idx 0–7)

#### PAT-006 — SUR/EDN/SKY Texture Pack
SUR: `#` family (solid)
EDN: `.` family (light)
SKY: `^` family (sky grain)
ANIM: slow PHASE drift (swap in alternates) + occasional Δx
PALETTE:
- SUR → Earth/Sand/Grass (idx 4/5/1)
- EDN → Greys (idx 18–21)
- SKY → Ice/Deep Sea/Cyan (idx 29/31/12)

## TELEtext50.ttf Mapping (Unicode Sextant Blocks)

Teletext50 is a tweak of Bedstead intended to emulate **SAA5050-era Teletext**. It relies on the Unicode **Symbols for Legacy Computing** block, including the **Block Sextant** characters.

### Sextant numbering convention (matches Unicode)
Each Teletext mosaic cell is split into **2×3** micro-cells numbered:

```
[ 1 2 ]
[ 3 4 ]
[ 5 6 ]
```

This matches the Unicode naming scheme: `Block Sextant-1356`, `Block Sextant-1234`, etc.

### Mapping rule (from our 6-bit mask)
We already defined our sextant mask bits as:

```
[ b0 b1 ]
[ b2 b3 ]
[ b4 b5 ]
```

So the translation is:
- b0 → sextant **1**
- b1 → sextant **2**
- b2 → sextant **3**
- b3 → sextant **4**
- b4 → sextant **5**
- b5 → sextant **6**

Algorithm:
1) Convert set bits → sextant numbers (ascending)
2) Concatenate numbers into a key (e.g. `1,3,5,6` → `1356`)
3) Look up the corresponding Unicode codepoint in the table below
4) Special cases:
   - mask `000000` → space `U+0020`
   - mask `111111` → full block `█ U+2588`
