**Schema Analysis:**

**sprite.schema.json** (248 lines):
- Required: `type`, `name`, `stats`, `meta`
- Naming: `^[A-Z][A-Z0-9-]*$` (UPPERCASE-HYPHEN)
- Stats: hp, hp_max, mp, mp_max, level, xp, attack, defense, speed, luck
- Inventory: gold, items (array), equipment (slots), capacity
- Status: state (alive/dead/unconscious/sleeping), effects, buffs
- Skills: id, name, mp_cost, power, level_required
- Meta: created, modified, scope (global/session/script/local), tags
- Example: HERO character with full stats/inventory/skills

**object.schema.json** (339 lines):
- Required: `type`, `name`, `category`, `meta`
- Naming: `^[A-Z][A-Z0-9-]*$` (UPPERCASE-HYPHEN)
- Categories: weapon, armor, shield, accessory, consumable, key-item, material, quest-item
- Properties: stackable, max_stack, weight, value, rarity, tradeable, droppable
- Stats: attack, defense, speed, hp_bonus, mp_bonus, luck
- Effects: hp_restore, mp_restore, status_cure, status_apply, buff_duration
- Requirements: level, stats (attack/defense/speed), class
- Display: display_name, description, icon (emoji), flavor_text
- Crafting: craftable, recipe (items+quantity), skill_required, level_required
- Examples: SWORD-IRON (weapon), POTION (consumable)

## Notes

### DOCS Command Structure

```
DOCS <command> [args] [options]

Commands:
  SEARCH <query>           # Search all content
  GUIDE <category>         # Knowledge guides
  DIAGRAM <name>           # SVG diagrams
  LEARN <adventure>        # Interactive learning
  REVIEW <name>            # Quality assessment ✅ COMPLETE
  REGEN <name> [options]   # AI regeneration ⚠️ PENDING
  HISTORY <name>           # Version history ⚠️ PENDING
  TYPES                    # List categories
  CONTINUE                 # Resume last guide

REGEN Options:
  --pro                    # Enhanced generation
  --strict                 # Strict citation mode
  --preview                # Show changes without applying
  --rollback              # Revert to previous version
```

### Variable System Naming Convention

**UPPERCASE-HYPHEN Format:**
- Pattern: `^[A-Z][A-Z0-9-]*$`
- Examples:
  - SPRITES: `HERO`, `PLAYER-1`, `NPC-GUARD-2`
  - OBJECTS: `SWORD-IRON`, `POTION`, `KEY-ITEM-1`
  - MISSIONS: `MISSION-WATER-FILTER-BUILD`
  - CHECKLISTS: `CHECKLIST-FIRE-SAFETY`

**Variable Access:**
```
# Read
GET $SPRITE.HERO.stats.hp
GET $OBJECT.POTION.effects.hp_restore

# Write
SET $SPRITE.HERO.stats.hp 75
SET $OBJECT.POTION.properties.value 100
```

### Archive System Integration

DOCS REGEN and HISTORY will use `.archive/` structure:

```

├── versions/
│   ├── guide_water_purification_v1_20251203_120000.md
│   ├── guide_water_purification_v2_20251203_143022.md
│   └── metadata.json
└── deleted/
```
