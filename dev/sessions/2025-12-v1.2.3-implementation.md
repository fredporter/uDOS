## Lessons Learned

### What Worked Well

1. **Parallel development:** Map layers + spatial data created simultaneously while workflow runs
2. **JSON-first approach:** Easy to validate, version control, and extend
3. **Existing map engine:** Layer support already built in, just needed data
4. **GeoJSON standard:** GitHub/web visualization without custom tools
5. **Workflow automation:** knowledge-expansion.upy handles generation end-to-end

### Challenges

1. **Workflow monitoring:** Background process, need better progress visibility
2. **GeoJSON size:** Full 129,600 cell export would be massive (defer to v1.2.4)
3. **Data completeness:** Only 4 cities in locations.json (need 200+ in future)
4. **Layer integration:** Map engine needs commands to switch/view layers
5. **Testing:** Need comprehensive tests before release

### Future Improvements

1. **v1.2.4:** Add 200+ cities to locations.json
2. **v1.2.4:** Create MAP command to switch layers, view TILE data
3. **v1.2.4:** Full GeoJSON export script (all cells, all layers)
4. **v1.2.5:** Integrate satellite layer with mesh networking
5. **v1.2.5:** Use planetary data for time sync in mesh
6. **v2.0.0:** Real-time weather API integration for cloud layer
