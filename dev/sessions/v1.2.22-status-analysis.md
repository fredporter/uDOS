## 🔧 Filename Convention Implementation Status

### Current Implementation

**Format Defined in Roadmap:**
```
YYYY-MM-DD-filename.ext              # Daily files
YYYY-MM-DD-HH-MM-SS-filename.ext     # Session files
YYYY-MM-DD-HH-MM-SS-sss-filename.ext # Instance files (milliseconds)
YYYY-MM-DD-HH-MM-SS-TILE-filename.ext # Location-specific files
```

**Actually Implemented:**

1. **Backup System** (`core/commands/backup_handler.py`)
   - Format: `{timestamp}_{original_filename}`
   - Example: `20251212_143045_user.json`
   - ✅ Timestamp present
   - ❌ Not ISO 8601 format (no dashes)
   - ❌ No TILE location

2. **OK MAKE Commands** (`core/commands/ok_handler.py`)
   - Workflow: `workflow_{timestamp}.upy`
   - SVG: `graphic_{timestamp}.svg`
   - Doc: `{topic}_{timestamp}.md`
   - Mission: `mission_{category}_{tile}_{timestamp}.upy` ✅ **TILE INCLUDED**
   - ✅ Timestamp present
   - ❌ Inconsistent format
   - ✅ Mission includes TILE

3. **Sandbox System** (`core/commands/sandbox_handler.py`)
   - Archive: `{timestamp}_{filename}`
   - ✅ Timestamp present
   - ❌ No ISO format
   - ❌ No TILE

4. **System Exports** (`core/commands/system_handler.py`)
   - Pattern export: `{timestamp}-error-patterns.json`
   - ✅ Timestamp + descriptive name
   - ❌ No ISO format
   - ❌ No TILE

### What's Missing

1. **Systematic ISO 8601 Format**
   - Need: `YYYY-MM-DD-HH-MM-SS` separator style
   - Current: `YYYYMMDD_HHMMSS` (varies)

2. **Optional TILE Location Field**
   - Need: Universal detection of user's current TILE
   - Need: `--location TILE` flag on file creation commands
   - Current: Only in OK MAKE MISSION

3. **Universal Application**
   - Need: All file creation commands support convention
   - Need: Helper function: `generate_filename(base, ext, include_location=False)`
   - Current: Each handler implements own timestamp logic

### Recommendation for v1.2.23

**Task 1: Universal Filename Helper** (~200 lines)

```python
# core/utils/filename_generator.py

def generate_filename(
    base_name: str,
    extension: str,
    include_date: bool = True,
    include_time: bool = False,
    include_location: bool = False,
    tile_code: Optional[str] = None
) -> str:
    """
    Generate standardized filename with optional date-time-location.
    
    Args:
        base_name: Base filename (e.g., "workflow", "backup")
        extension: File extension (e.g., ".upy", ".json")
        include_date: Include YYYY-MM-DD prefix
        include_time: Include HH-MM-SS after date
        include_location: Include TILE code
        tile_code: Explicit TILE code (or detect from config)
    
    Returns:
        Formatted filename following uDOS convention
    
    Examples:
        >>> generate_filename("backup", ".json", include_date=True, include_time=True)
        "2025-12-12-14-30-45-backup.json"
        
        >>> generate_filename("mission", ".upy", include_time=True, include_location=True, tile_code="AA340")
        "2025-12-12-14-30-45-AA340-mission.upy"
    """
    parts = []
    
    if include_date:
        now = datetime.now()
        parts.append(now.strftime("%Y-%m-%d"))
        
        if include_time:
            parts.append(now.strftime("%H-%M-%S"))
    
    if include_location and tile_code:
        parts.append(tile_code)
    
    parts.append(base_name)
    
    return "-".join(parts) + extension
```

**Integration Points:**
- Update 7 handlers: backup, ok, sandbox, archive, system, file, workflow
- Add `--dated`, `--timed`, `--located` flags to NEW/COPY/BACKUP commands
- Config option: `auto_timestamp_files` (boolean)
- Config option: `auto_location_files` (boolean)
- TILE detection: Use `self.config.get('current_tile')` or detect from timezone
