# Sandbox to Memory Migration - v1.2.12

**Date:** December 5, 2025
**Version:** v1.2.12
**Status:** ✅ Complete

## Summary

Deprecated `sandbox/` directory in favor of unified `memory/` workspace structure. All user data, logs, and configuration files have been migrated to the new paths.

## Files Migrated

### User Data
- `sandbox/user/user.json` → `memory/bank/user/user.json`
- `sandbox/user/*.db` → `memory/bank/data/*.db`
- `sandbox/user/*.json` → `memory/bank/user/*.json`

### Logs
- `sandbox/logs/*` → `memory/logs/*`
- `sandbox/logs/sessions/*` → `memory/logs/sessions/*`
- All log files preserved with timestamps

### Archived
- `sandbox/user/` → `sandbox/.archive/user/`
- `sandbox/logs/` → `sandbox/.archive/logs/`

## New Directory Structure

```
memory/
├── logs/                   # All system logs (was sandbox/logs)
│   ├── sessions/          # Session recordings
│   └── .archive/          # Old logs (auto-managed)
├── bank/                   # Banking/config system
│   ├── user/              # User data (was sandbox/user)
│   │   ├── user.json      # Main user config
│   │   └── *.json         # Other user files
│   ├── system/            # System config files
│   │   └── *.json         # System settings
│   ├── data/              # Application databases
│   │   └── *.db           # SQLite databases (XP, etc.)
│   ├── private/           # Private transactions
│   └── barter/            # Barter system
├── shared/                # Community/shared content
│   ├── groups/            # Group workspaces (was sandbox/groups)
│   │   └── */             # Per-group folders
│   └── public/            # Public shares (was sandbox/user/poke_shares.json)
├── ucode/                 # Core distributable scripts (tracked)
├── workflows/             # Workflow automation (was sandbox/workflow)
│   ├── missions/          # Mission scripts
│   ├── checkpoints/       # Auto-saved state
│   ├── state/             # Current execution
│   └── .archive/          # Old workflows
├── missions/              # Mission management (was sandbox/workflow/missions)
└── drafts/                # Draft content
    ├── svg/               # SVG drafts (was sandbox/drafts/svg)
    ├── ascii/             # ASCII art
    └── teletext/          # Teletext pages
```

## Command Changes

### CLEAN Command
**Before (v1.2.11):**
```
CLEAN                    # Review files in sandbox/
```

**After (v1.2.12):**
```
CLEAN                    # Show deprecation notice + archive system info
CLEAN --scan             # View archive health across workspace
CLEAN --purge [days]     # Remove old backups
```

### RESOURCES Command
**Before:**
```
💾 Disk Space:
  ✅ Sandbox: 45.2 MB
  ✅ System: 512000/1024000 MB (50%)
```

**After:**
```
💾 Disk Space:
  ✅ Memory: 45.2 MB
  ✅ System: 512000/1024000 MB (50%)
```

### Path Examples
**Before:**
```
WORKFLOW LOAD_JSON sandbox/data/config.json
SPRITE LOAD WARRIOR sandbox/user/warrior.json
```

**After:**
```
WORKFLOW LOAD_JSON memory/bank/data/config.json
SPRITE LOAD WARRIOR memory/bank/user/warrior.json
```

## Breaking Changes

### For Users
- **None** - Migration is automatic
- Old files preserved in `sandbox/.archive/` for 90 days
- All commands work with new paths automatically

### For Developers/Extension Authors
Update hardcoded paths in custom extensions:
```python
# Old
user_file = "sandbox/user/user.json"
log_dir = "sandbox/logs"
data_file = "sandbox/user/data.db"

# New
user_file = "memory/bank/user/user.json"
log_dir = "memory/logs"
data_file = "memory/bank/data/data.db"
```

## Rollback Procedure

If issues arise:
```bash
# 1. Restore old files
cp -r sandbox/.archive/user/* sandbox/user/
cp -r sandbox/.archive/logs/* sandbox/logs/

# 2. Revert code changes
git checkout v1.2.11

# 3. Restart uDOS
./start_udos.sh
```

Files in `sandbox/.archive/` are preserved for 90 days.
