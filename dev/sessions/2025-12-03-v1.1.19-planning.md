#### Phase 1: Gap Analysis (Steps 1-5)

**Objective:** Identify missing topics systematically

**Implementation:**
```python
# Pseudo-workflow structure
PHASE "Gap Analysis"
  FOR category IN [water, fire, shelter, food, navigation, medical]:
    LOAD existing_guides FROM knowledge/{category}/
    LOAD topic_master_list FROM core/data/knowledge_topics.json

    COMPARE existing_guides WITH master_list
    GENERATE gap_report WITH:
      - Missing topics (critical, important, nice-to-have)
      - Coverage percentage
      - Priority ranking

    SAVE gap_report TO memory/workflows/state/gaps_{category}.json
  END FOR

  GENERATE consolidated_report
  PROMPT user FOR category selection
END PHASE
```

#### Phase 2: Content Generation (Steps 6-10)

**Objective:** Batch generate articles using Gemini API + GENERATE command

**Workflow:**
```python
PHASE "Content Generation"
  SELECT category FROM gap_report
  SELECT topics[0:10]  # Batch of 10

  FOR topic IN selected_topics:
    # Use existing GENERATE system
    EXECUTE GENERATE GUIDE {topic} --category {category} --style survival

    # Output: memory/drafts/knowledge/{category}/{topic}.md
    WAIT for completion

    LOG generation_stats:
      - Topic: {topic}
      - Word count
      - Generation time
      - API cost
  END FOR

  CHECKPOINT save_batch_state
END PHASE
```

**Integration Points:**
- Use existing `GENERATE` command (enhanced in v1.1.15)
- Leverage Nano Banana style guides (technical-kinetic, hand-illustrative)
- Apply survival-specific prompts from `survival_prompts.json`

**Quality Criteria:**
- Minimum 500 words
- 3+ sections (introduction, steps/details, tips/warnings)
- At least 1 diagram/illustration
- Citations/references where applicable
- Markdown formatting compliance

#### Phase 3: Quality Review (Steps 11-15)

**Objective:** Automated quality assessment using REVIEW command

**Workflow:**
```python
PHASE "Quality Review"
  FOR article IN generated_batch:
    # Use DOCS REVIEW from v1.1.17
    EXECUTE DOCS REVIEW {article}

    GET scores:
      - Completeness (0-100)
      - Clarity (0-100)
      - Accuracy (0-100)
      - Usefulness (0-100)
      - Overall (average)

    IF overall < 85:
      # Needs improvement
      GET improvement_suggestions
      EXECUTE DOCS REGEN {article} WITH suggestions
      RE-REVIEW regenerated_article
    END IF

    SAVE review_report TO state/reviews/{article}.json
  END FOR

  CALCULATE approval_rate = (articles >= 85) / total_articles
  REPORT approval_rate
END PHASE
```

**Success Threshold:**
- Target: 85%+ overall score
- Minimum: 80% to proceed
- Below 80%: Manual review required

#### Phase 4: Validation (Steps 16-20)

**Objective:** Standards compliance checking

**Checks:**
1. **Markdown Structure:**
   - Valid heading hierarchy (H1 → H2 → H3)
   - Code blocks properly fenced
   - Lists formatted correctly
   - Links valid

2. **Content Standards:**
   - Safety warnings present (if applicable)
   - Step-by-step procedures clear
   - Materials list included
   - Difficulty level specified

3. **Metadata:**
   - Category assigned
   - Tags present
   - Related guides linked
   - Last updated timestamp

**Workflow:**
```python
PHASE "Validation"
  FOR article IN approved_articles:
    RUN markdown_linter(article)
    CHECK content_standards(article)
    VERIFY metadata_complete(article)

    IF all_checks_pass:
      MARK article AS validated
    ELSE:
      LOG validation_errors
      PROMPT user FOR fix_or_skip
    END IF
  END FOR

  GENERATE validation_report
END PHASE
```

#### Phase 5: Commit (Steps 21-25)

**Objective:** Move approved articles to knowledge bank

**Workflow:**
```python
PHASE "Commit"
  FOR article IN validated_articles:
    SOURCE = memory/drafts/knowledge/{category}/{topic}.md
    DEST = knowledge/{category}/{topic}.md

    # Safety checks
    IF file_exists(DEST):
      PROMPT user FOR overwrite_confirmation
    END IF

    MOVE SOURCE TO DEST
    GIT add {DEST}

    LOG commit_action:
      - File: {topic}.md
      - Category: {category}
      - Timestamp: {now}
      - Review score: {score}
  END FOR

  GIT commit -m "Knowledge: Added {count} {category} guides"

  UPDATE knowledge bank index
  REGENERATE category README files
END PHASE
```

**Post-Commit:**
- Update `knowledge/README.md` statistics
- Regenerate category index files
- Update wiki documentation
- Archive workflow state
