## 📊 Current State Analysis

### Existing Systems

**1. Calendar Tasks** (`memory/bank/user/tasks.json`)
- Location: `core/commands/calendar_handler.py`
- Storage: Single JSON file
- Features: id, description, status, priority, due_date, progress
- Commands: TASK LIST/ADD/DONE/DEL

**2. Workflows** (`memory/workflows/missions/*.upy`)
- Location: `core/commands/workflow_handler.py`
- Storage: .upy scripts in missions folder
- Features: LOG, CHECKPOINT, RUN_PYTHON, status tracking
- Commands: WORKFLOW commands for automation

**3. Checklists** (`knowledge/checklists/*.json`)
- Location: `core/commands/checklist_handler.py`  
- Storage: JSON files with nested items
- Features: category, items, completion tracking, reset
- Commands: CHECKLIST LIST/LOAD/COMPLETE/STATUS

**4. Missions/Projects** (`memory/workflows/missions/`)
- Location: uPY scripts with embedded objectives
- Storage: Standalone .upy files
- Features: Business intelligence gathering, location-based workflows
- Example: `discover-sydney-music-venues.upy`

### Integration Gaps

1. **No Unified Model**: Tasks ≠ Checklists ≠ Missions ≠ Workflows
2. **Separate Storage**: 3+ different locations for task-like entities
3. **No Cross-Reference**: Can't link task → checklist → mission → workflow
4. **Inconsistent Filenames**: Mix of formats (no uDOS ID standard)
5. **Limited Archiving**: No systematic completed task/mission archiving
6. **No Relationship Tracking**: Tasks can't belong to missions/projects

## 🎯 Unified Task System Architecture

### Core Concept: Everything is a Task

```
Task (base entity)
├── Simple Task       → Calendar entry
├── Checklist Item    → Part of guided process  
├── Workflow Step     → Automated execution
└── Mission Objective → Project deliverable
```

## ⚠️ Breaking Changes & Migration

### User Impact

**Before v1.2.23:**
```
memory/bank/user/tasks.json         # Calendar tasks
knowledge/checklists/*.json         # Checklists
memory/workflows/missions/*.upy     # Missions (generic names)
```

**After v1.2.23:**
```
memory/workflows/tasks/unified_tasks.json              # ALL tasks
memory/checklists/*.json                               # User checklists
memory/workflows/missions/20251213-HHMMSS-name.upy    # Timestamped missions
```

### Migration Process

**Automatic (first run after v1.2.23):**
1. Detect old task/checklist files
2. Prompt: "Migrate to unified task system? (y/n)"
3. Create backups in .archive/migration/
4. Convert and merge into unified_tasks.json
5. Rename mission files with timestamps
6. Archive old files with deprecation notice

**Manual (if needed):**
```bash
python dev/tools/migrate_to_unified_tasks.py
python dev/tools/rename_distributable_files.py
```

### Rollback Plan

If migration fails:
1. Restore from .archive/migration/
2. Revert to v1.2.22
3. Report issue to GitHub
