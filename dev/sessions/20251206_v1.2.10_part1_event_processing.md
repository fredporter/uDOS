### Pattern-Based Routing

```python
# Simple platform match
{'platform': 'github'}

# Event type with regex
{'event_type': r'code\..*'}

# Source filtering
{'platform': 'github', 'source': {'repository': 'fredporter/uDOS'}}

# Multiple event types
{'event_type': ['code.pushed', 'code.pull_request.opened']}
```

### Async Handler Support

```python
router = EventRouter()

async def handler(event):
    # Process event
    return {'success': True}

router.register_route(
    name='my_handler',
    pattern={'event_type': 'code.pushed'},
    handler=handler,
    priority=50
)

results = await router.route(event)
```

## Architecture Decisions

1. **Async-First Design:** All routing and handler execution is async for non-blocking event processing

2. **Pattern-Based Routing:** Flexible regex patterns allow complex event matching without hardcoding routes

3. **Priority System:** Handlers execute in priority order (0-100, higher first) for predictable behavior

4. **Event History:** Configurable history retention for debugging and analytics

5. **Platform Agnostic:** Unified event format enables cross-platform workflows

6. **Statistics Tracking:** Built-in metrics for monitoring and optimization

7. **Error Resilience:** Continue processing on handler errors (configurable)

8. **Script Integration:** Direct uCODE script execution on events via interpreter
