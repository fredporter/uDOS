# uDOS Development Philosophy

## We Are Building v1.0

uDOS is a foundational system currently at v1.0.4.1, working toward our first stable release. The progression through "versions" during development has been helpful for organizing work, but we need to be clear about our actual maturity level.

## Language Principles

### Use These Terms
- **foundational** - We're building the foundation
- **core** - Essential, basic functionality
- **essential** - Required, fundamental features
- **clean** - Simple, uncluttered design
- **simple** - Easy to understand and use
- **lean** - Efficient, minimal overhead
- **fast** - Quick response, optimized performance

### Avoid These Terms in Development Context
- **latest** - Implies established product with updates
- **updated** - Suggests improving existing stable version
- **enhanced** - Implies adding to mature features
- **extended** - Suggests expanding finished functionality
- **legacy** - Has specific meaning in uDOS workflow (see below)

## The "Legacy" Term in uDOS

In uDOS, "legacy" has a specific, positive meaning in our workflow system:

**Move → Milestone → Mission → Legacy**

- **Legacy** is the endpoint goal - what each installation strives to leave for the next user
- **Legacy** means "worth keeping" and "valuable contribution"
- **Legacy** is something positive to achieve, not something old to avoid

### Do NOT use "legacy" for:
- Old code that needs updating
- Outdated systems or practices
- Deprecated functionality
- Technical debt

This prevents confusion between our workflow concept and typical development terminology.

## Documentation Approach

### Living Documents
- Remove version numbers from filenames
- Write docs that evolve with the system
- Focus on current capabilities, not "new" features
- Create snapshots in `/dev/docs/` when needed for reference

### Foundational Messaging
- Emphasize core capabilities over feature lists
- Focus on reliability and compatibility
- Highlight simplicity and clean design
- Avoid implying advanced or premium features

## Version Management

### Current Reality
- **System Version**: 1.0.4.1 (building toward 1.0 stable)
- **Development Stage**: Active foundation building
- **Documentation**: Living documents without version numbers
- **Philosophy**: Simple, lean, fast foundational system

### Moving Forward
- Continue building solid foundations
- Focus on core functionality over feature expansion
- Maintain clear, simple documentation
- Prepare for genuine 1.0 stable release when ready

This approach keeps us grounded in reality while building something truly solid and dependable.
