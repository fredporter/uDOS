## Problem Identified

The uDOS repository was bloated with **build artifacts and local development files** that should never be tracked in git:

- **6,702 node_modules files** (87% of repo!) - Users should run `npm install` to build these
- **141 __pycache__ files** - Python bytecode generated at runtime
- **101 dev/sessions files** - Local development session notes (not user documentation)

### Phase 2: Separate Dev Mode to Submodule (25,595 lines removed)
```bash
# 1. Initialize dev/ as separate git repository
cd dev/
git init
git add -A
git commit -m "Initial commit: uDOS development tools"

# 2. Create GitHub repository uDOS-dev
# Created at: https://github.com/fredporter/uDOS-dev

# 3. Push to GitHub (resolved merge conflicts)
git remote add origin https://github.com/fredporter/uDOS-dev.git
git pull origin main --allow-unrelated-histories
git push -u origin main

# 4. Add dev/ as submodule to main repo
cd ..
git submodule add https://github.com/fredporter/uDOS-dev.git dev
git commit -m "Refactor: Move dev/ to submodule"
git push origin main
```

### Phase 3: Update .gitignore (v1.2.21)
New structure with clear categories:
- **CRITICAL: Build Artifacts & Dependencies (NEVER TRACK)**
- **Development Sessions (NEVER TRACK)** - `dev/sessions/` now ignored
- **Node.js** - `node_modules/`, `**/ node_modules/`, `package-lock.json`
- **Python bytecode** - `__pycache__/`, `**/__pycache__/`, `*.pyc`
- **Build outputs** - `dist/`, `build/`, `*.vsix`

### 3. File Placement Philosophy

**✅ TRACKED (Distribution-ready content):**
- Core system (`core/`, `knowledge/`)
- Production extensions (`extensions/play/`, `extensions/assistant/`)
- Distributable scripts (`memory/ucode/{tests,stdlib,examples,adventures}`)
- Documentation (`wiki/`, `dev/roadmap/`, `README` files)

**❌ NEVER TRACKED (Local-only):**
- Node modules (users run `npm install`)
- Python bytecode (generated at runtime)
- User data (`memory/workflows/`, `memory/missions/`)
- Development sessions (`dev/sessions/`)
- Virtual environments (`.venv/`)
- Build artifacts (`dist/`, `*.vsix`)

## Benefits

1. **Cleaner Repository**: 84.5% fewer files tracked
2. **Faster Clones**: Less data to download (eventually, after git history cleanup)

4. **Standard Practice**: Users install dependencies locally (npm, pip)
5. **Better .gitignore**: Clear categories, no ambiguity

## User Installation Process

After cloning, users now need to:

```bash
# 1. Clone repository
git clone https://github.com/fredporter/uDOS.git
cd uDOS

# 2. Install Python dependencies
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# 3. Install VS Code extension dependencies (optional)
cd extensions/vscode
npm install
cd ../..

# 4. Run uDOS
./start_udos.sh
```

## Future Optimizations

To reduce the **packed size** (currently 103.73 MiB), we could:

1. **Git History Cleanup** - Use `git filter-branch` or BFG Repo-Cleaner to purge old node_modules from history
   - **Risk**: Rewrites git history, breaks existing clones
   - **Benefit**: Could reduce to ~10-20 MiB
   - **Status:** Deferred to v1.3.0+

2. **Separate VS Code Extension** - Move `extensions/vscode/` to separate repo
   - **Benefit**: Core uDOS becomes tiny (~5 MB)
   - **Trade-off**: Users install extension separately
   - **Status:** Consider for v1.3.0+

3. ✅ **Dev Mode as Separate Project** - COMPLETE in v1.2.21
   - **Implementation**: Git submodule at https://github.com/fredporter/uDOS-dev
   - **Benefit**: Development tools separate from production (36 files optional)
   - **Result:** Users get 1,331 files, contributors get 1,367 files
