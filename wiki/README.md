## 🤝 Contributing to the Wiki

### Adding New Pages
1. Create `NewPage.md` in the `wiki/` directory
2. Write content using GitHub Flavored Markdown
3. Add navigation links to this README and `_Sidebar.md`
4. Run `./deploy_wiki.sh` to publish to GitHub

### Improving Existing Pages
1. Edit `.md` files directly in `wiki/` directory
2. Follow the **[Style Guide](Style-Guide.md)**
3. Test links and formatting locally
4. Submit pull request with clear description

### Documentation Standards
- ✅ **Plain language**: No jargon unless necessary
- ✅ **Working examples**: Every concept has examples
- ✅ **Context provided**: Explain WHY, not just HOW
- ✅ **Complete workflows**: Show full task sequences
- ✅ **Progressive disclosure**: Start simple, add complexity
- ✅ **Cross-referenced**: Link to related pages

See **[Contributing](Contributing.md)** for complete guidelines.

## 📝 Markdown Guidelines

### Formatting Standards
- Use `#` for page title (one per page)
- Use `##` for major sections
- Use `###` for subsections
- Include code blocks with syntax highlighting
- Add tables for comparisons
- Use emojis sparingly for visual emphasis
- Cross-reference: `[Link Text](Page-Name.md)`

### Example Structure
```markdown
# Page Title

Brief introduction to the topic.

---

## First Major Section

Content here.

### Subsection

More details.

## Second Major Section

More content.

---

## See Also

- [Related Page 1](Page1.md)
- [Related Page 2](Page2.md)
```
