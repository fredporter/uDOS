# SVG Example Gallery

Visual examples of SVG diagrams generated by the uDOS SVG Graphics Extension. Each example shows the command used, description of output, and typical use cases.

## Quick Reference

| Style | Best For | Output Characteristics |
|-------|----------|----------------------|
| **lineart** | Clean diagrams, icons | Black lines, no fill, minimal |
| **blueprint** | Technical docs, plans | Blue grid, precise, annotated |
| **sketch** | Informal notes, brainstorming | Hand-drawn look, organic |
| **isometric** | 3D views, architecture | Isometric projection, depth |

## Lineart Style Examples

### Basic Water Filter

**Command:**
```
SVG water filtration layers --style lineart
```

**Output Characteristics:**
- Clean black lines (stroke-width: 1)
- No fills (transparent shapes)
- Simple geometric shapes
- Clear layer separation
- Size: ~445 chars

**Typical Elements:**
```xml

  <text x="310" y="180" font-size="12">Sand</text>
  <text x="310" y="230" font-size="12">Charcoal</text>
</svg>
```

**Use Cases:**
- Quick diagrams for printing
- Icon creation
- Minimalist technical illustrations
- Screen-friendly visuals

## Blueprint Style Examples

### Shelter Construction

**Command:**
```
SVG A-frame shelter construction plan --style blueprint
```

**Output Characteristics:**
- Blue color scheme (#1E3A8A, #3B82F6, #93C5FD)
- Grid background pattern
- Precise measurements
- Detailed annotations
- Size: ~1875 chars (most detailed)

**Typical Elements:**
```xml
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 800 600">
  <defs>
    <pattern id="grid" width="20" height="20" patternUnits="userSpaceOnUse">

    </pattern>
  </defs>
  <rect width="800" height="600" fill="url(#grid)"/>
  <polygon points="400,150 250,450 550,450" stroke="#1E3A8A" fill="none" stroke-width="2"/>
  <line x1="400" y1="150" x2="400" y2="450" stroke="#3B82F6" stroke-dasharray="5,5"/>
  <text x="410" y="300" font-size="10" fill="#1E3A8A">8 ft</text>
  <text x="300" y="470" font-size="10" fill="#1E3A8A">A-FRAME SHELTER</text>
</svg>
```

**Use Cases:**
- Technical documentation
- Construction plans
- Detailed guides
- Engineering diagrams

## Sketch Style Examples

### Campfire Setup

**Command:**
```
SVG campfire setup with rocks and kindling --style sketch
```

**Output Characteristics:**
- Hand-drawn appearance
- Organic, irregular lines
- Warm earth tones (#8B4513, #D2691E, #F4A460)
- Informal feel
- Stroke-width: 2 (thicker)

**Typical Elements:**
```xml

  <!-- Rocks (irregular shapes) -->
  <ellipse cx="350" cy="450" rx="20" ry="15" stroke="#8B4513" fill="#D2691E" opacity="0.7"/>
  <ellipse cx="450" cy="450" rx="18" ry="16" stroke="#8B4513" fill="#D2691E" opacity="0.7"/>

  <!-- Kindling (crossed lines) -->
  <line x1="380" y1="380" x2="420" y2="420" stroke="#F4A460" stroke-width="2"/>
  <line x1="420" y1="380" x2="380" y2="420" stroke="#F4A460" stroke-width="2"/>

  <!-- Flame shapes (wavy) -->
  <path d="M 400 350 Q 390 320 400 300 Q 410 320 400 350" stroke="#D2691E" fill="#F4A460" opacity="0.5"/>

  <text x="350" y="520" font-size="14" font-family="cursive">Campfire Setup</text>
```

## Size Comparison

Typical SVG file sizes by style:

| Style | Avg Size | Complexity | Elements |
|-------|----------|------------|----------|
| lineart | 445 chars | Low | 5-10 shapes |
| sketch | 600 chars | Medium | 10-15 shapes |
| isometric | 800 chars | Medium-High | 15-20 shapes |
| blueprint | 1875 chars | High | 20+ shapes + patterns |

All sizes well within uDOS target (< 50KB for text files).

## Integration Examples

### With GUIDE Command

```bash
# Generate knowledge guide and diagram together
GUIDE water purification
SVG water purification process --style blueprint --save purification-process.svg

# Result: Text guide + visual diagram in same folder
```

### With BANK Command

```bash
# Search knowledge and create diagram
BANK fire methods
SVG fire building methods comparison --style sketch

# Result: Diagram based on knowledge bank content
```

### With DRAW Command

```bash
# Create both ASCII and SVG
DRAW fire triangle
SVG fire triangle showing components --style lineart

# Result: ASCII art (terminal) + SVG (file)
```

## Style Selection Guide

### When to Use Each Style

**Lineart:**
- ✅ Clean, professional diagrams
- ✅ Icons and symbols
- ✅ Screen display
- ✅ Minimal printing
- ❌ Not for: Complex technical docs

**Blueprint:**
- ✅ Technical documentation
- ✅ Engineering plans
- ✅ Precise measurements
- ✅ Professional presentations
- ❌ Not for: Casual notes, quick sketches

**Sketch:**
- ✅ Field notes
- ✅ Brainstorming
- ✅ Informal guides
- ✅ Natural subjects (plants, animals)
- ❌ Not for: Formal documentation, precision work

**Isometric:**
- ✅ 3D visualizations
- ✅ Architecture
- ✅ System layouts
- ✅ Spatial relationships
- ❌ Not for: Flat diagrams, simple concepts

---

## Tips for Best Results

1. **Keep descriptions concise** (5-15 words optimal)
2. **Specify key elements** ("showing layers" vs just "filter")
3. **Match style to purpose** (blueprint for technical, sketch for informal)
4. **Save important diagrams** (use `--save` parameter)
5. **Test without AI** (fallback templates work well)
6. **Combine with knowledge** (use GUIDE, then SVG)
7. **Batch similar diagrams** (same style for consistency)
