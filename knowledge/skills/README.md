## Content Guidelines

### ✅ Include
- **Beginner-friendly** introductions
- **Progressive** skill development paths
- **Practice exercises** with solutions
- **Resources** for continued learning
- **Common mistakes** and how to avoid them
- **Free/open-source** tools when possible

### ❌ Exclude
- Overly advanced topics without prerequisites
- Tool-specific tutorials that will quickly become outdated
- "Get rich quick" schemes
- Unrealistic timelines ("master programming in 30 days")
- Proprietary software requirements (prefer open alternatives)

## Skills Philosophy

**Human-centric skill development means**:
- Learning for joy and capability, not just career
- Mastery takes time—be patient with yourself
- Fundamentals matter more than trendy techniques
- Teaching others reinforces your own learning
- Skills compound—each new skill makes others easier
- Progress, not perfection

## Learning Paths

### Beginner Path
1. **Choose one skill** to focus on (don't spread too thin)
2. **Learn fundamentals** before advanced techniques
3. **Practice daily** (even 15 minutes counts)
4. **Build small projects** to apply knowledge
5. **Share progress** for feedback and motivation

### Intermediate Path
1. **Deepen fundamentals** with deliberate practice
2. **Study others' work** in your chosen skill
3. **Take on challenges** slightly beyond current ability
4. **Teach beginners** what you've learned
5. **Explore adjacent skills** that complement your main focus

### Advanced Path
1. **Develop personal style** and approach
2. **Contribute to community** (open source, tutorials, etc.)
3. **Combine skills** in unique ways
4. **Mentor others** systematically
5. **Never stop learning** fundamentals

## Contributing

When adding skills knowledge:
1. **Start simple** - Assume no prior knowledge
2. **Include exercises** - Practice makes progress
3. **Link resources** - Books, tutorials, tools
4. **Show examples** - Demonstrate concepts concretely
5. **Update regularly** - Skills and tools evolve
